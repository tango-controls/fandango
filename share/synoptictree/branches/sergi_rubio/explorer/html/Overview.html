1 - Getting started:

Device Tree is an application to watch over a list of attributes and commands. There are 2 different modes in this application:

  • Administrator mode in which you can parameter the application and save your preferences.
    To use this mode, launch application with no argument.
  • User mode in which you watch over your attributes and commands defined in a device tree file.
    To use this mode, launch application with the path of your device tree file as argument.

See Device Tree Manual for more informations.

2 - The packages:

There are two packages in this application:

  • explorer that takes care of the main work
  • explorer.ui that takes care of the User Interface

Browse these packages to see the application architecture

3 - The Different ways to start Device Tree:

3.1 - The "Stand Alone" starting:

This is the most used one. Just launch the devicetree script (main class is explorer.Main), following instructions in the 1 - Getting started: section.

3.2 - The "Not Stand Alone" starting:

This case comes when you want to start Device Tree through another application.
Let's see how you can start the two different modes as "Not Stand Alone":

  • Administrator mode:
    To use this mode, instanciate a new explorer.AdminMain()
  • User mode:
    To use this mode, instanciate a new explorer.ViewerMain(filename), where filename is a String that represents the path of the device tree file you want to start your user mode with.
